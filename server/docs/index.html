API Documentation

Everything you need to integrate NyxStatus into your workflow.

Authentication

Include your API key in the Authorization header:

Authorization: Bearer nxs_live_your_api_key

Monitors

GET /api/v1/monitors

List all monitors in your organization.

Response:

{ "monitors": [ { "id": "mon_abc123", "name": "Production API", "url": "https://api.acme.com/health", "status": "up", "uptime_30d": 99.99, "avg_response_ms": 45 } ] }

POST /api/v1/monitors

Create a new monitor.

Request body:

{ "name": "My API", "url": "https://api.example.com/health", "type": "https", "interval": 30, "alert_channels": ["email", "slack"] }

GET /api/v1/monitors/:id

Get a single monitor with detailed stats.

PUT /api/v1/monitors/:id

Update a monitor's configuration.

DELETE /api/v1/monitors/:id

Delete a monitor.

Incidents

GET /api/v1/incidents

List all incidents. Filter by status: open, resolved, all.

Query params:

?status=open&limit=10&page=1

POST /api/v1/incidents

Create a new incident.

Request body:

{ "title": "API Degradation", "status": "investigating", "message": "Elevated error rates detected.", "affected_monitors": ["mon_abc123"] }

POST /api/v1/incidents/:id/updates

Add an update to an incident.

Request body:

{ "status": "resolved", "message": "Root cause identified and fix deployed." }

Status Pages

GET /api/v1/status-pages

List your status pages.

POST /api/v1/status-pages

Create a public status page.

Request body:

{ "name": "Acme Corp Status", "subdomain": "acme", "monitors": ["mon_abc123", "mon_def456"] }

Webhooks

NyxStatus can send webhook notifications for the following events:

monitor.down — Monitor detected as down
monitor.up — Monitor recovered
monitor.degraded — Response time exceeds threshold
incident.created — New incident created

Rate Limits

API requests are rate limited per plan:

Hobby 60 req/min
Pro 300 req/min
Business 1000 req/min
